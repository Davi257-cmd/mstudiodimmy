## 1. Visão Geral do Produto

O M Studio Dimmy é um site institucional premium que tem como objetivo principal estabelecer a autoridade e talento de Dimmy Marinho no setor de beleza e estética. O site deve transmitir exclusividade, modernidade, confiança e acessibilidade, funcionando como uma vitrine profissional que facilita o contato e agendamento de serviços, reforça parcerias com marcas premium (especialmente JO's Professional) e serve como plataforma para evidenciar o trabalho e expertise do profissional.

## 2. Objetivos do Projeto

- Estabelecer autoridade e credibilidade de Dimmy Marinho no mercado de beleza
- Facilitar o agendamento de serviços e orçamentos através de integração com WhatsApp
- Apresentar portfólio profissional com galeria de resultados e feed do Instagram
- Reforçar parcerias com marcas premium, destacando especialmente JO's Professional
- Fornecer experiência mobile-first com carregamento rápido e SEO otimizado
- Transmitir valores de excelência, educação constante, inovação e personalização

## 3. Público-Alvo

| Segmento | Características | Necessidades |

## 5. Requisitos Técnicos

### 5.1 Stack Tecnológica
- **Frontend**: React 18+ com TypeScript para type safety
- **Build Tool**: Vite para desenvimento rápido e otimização
- **Estilização**: Tailwind CSS para design responsivo e consistente
- **Estado Global**: Zustand para gerenciamento leve de estado
- **Roteamento**: React Router DOM para navegação SPA
- **Ícones**: Lucide React ou React Icons para consistência visual
- **Animações**: Framer Motion ou CSS transitions para interações suaves

### 5.2 Integrações Externas
- **WhatsApp Business API**: wa.me/5585996163468 para agendamentos
- **Instagram Basic Display API**: @mstudiodimmy para feed integrado
- **Google Maps Embed API**: Localização Rua Oliveira Sobrinho, Bonsucesso
- **Google Analytics 4**: Tracking de eventos e conversões
- **Meta Pixel**: Remarketing e análise de tráfego

### 5.3 Requisitos de Performance
- **Core Web Vitals**: LCP < 2.5s, FID < 100ms, CLS < 0.1
- **Mobile First**: Design responsivo com breakpoint mobile-first
- **Lazy Loading**: Imagens e componentes carregados sob demanda
- **Code Splitting**: Divisão de bundles por rota
- **Image Optimization**: WebP format, responsive images, lazy loading
- **Caching Strategy**: Service worker para funcionalidade offline parcial

### 5.4 Requisitos de Acessibilidade
- **WCAG 2.1 AA**: Conformidade com diretrizes de acessibilidade
- **Alt Text**: Todas as imagens com descrições alternativas
- **Keyboard Navigation**: Navegação completa via teclado
- **Screen Reader**: Estrutura semântica com ARIA labels
- **Color Contrast**: Razão mínima 4.5:1 para texto normal, 3:1 para texto grande
- **Focus Indicators**: Estados de foco visíveis em todos elementos interativos

## 7. Requisitos de Design

### 7.1 Paleta de Cores
- **Primárias**: Preto (#000000), Branco (#FFFFFF)
- **Neutras**: Tons de cinza (#F5F5F5, #E5E5E5, #A0A0A0, #404040)
- **Destaques**: Dourado/Sépia (#D4AF37, #CD853F), Magenta (#FF00FF, #C2185B)

### 7.2 Tipografia
- **Títulos**: Fonte elegante sem serifa (inspiração fashion/beauty)
